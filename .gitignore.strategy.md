# 📋 Git Ignore 策略说明

## 推荐做法：根目录统一管理 ✅

对于Monorepo项目，**推荐只在根目录放置一个.gitignore文件**，统一管理所有忽略规则。

### 原因：

1. **简化管理** - 只有一个地方需要维护
2. **避免冲突** - 多个.gitignore可能导致规则冲突
3. **清晰可见** - 项目级别的忽略策略一目了然
4. **工具友好** - 大多数工具都支持从根目录读取配置

### 特殊情况需要子包.gitignore：

| 场景                | 是否需要子包.gitignore | 示例                 |
| ------------------- | ---------------------- | -------------------- |
| 通用Monorepo        | ❌ 不需要              | 本项目               |
| 子包发布为独立npm包 | ❌ 不需要              | 本项目               |
| 子包有特殊构建工具  | ⚠️ 可选                | 如子包使用特定编译器 |
| 子包有敏感配置文件  | ✅ 需要                | 子包特定的密钥文件   |

## 本项目的.gitignore结构

```
panther-expo/
├── .gitignore              ← ✅ 统一的忽略配置
├── .gitignore.strategy.md  ← 📖 本说明文档
└── packages/
    ├── utils/              ← ❌ 不需要.gitignore
    ├── theme/              ← ❌ 不需要.gitignore
    ├── core/               ← ❌ 不需要.gitignore
    └── ui/                 ← ❌ 不需要.gitignore
```

## 根目录.gitignore覆盖范围

```gitignore
# 1. 全局依赖
node_modules/

# 2. 所有包的构建输出
packages/*/dist/
packages/*/build/

# 3. 测试覆盖率
coverage/

# 4. 环境变量
.env

# 5. 编辑器配置
.vscode/
.idea/

# 6. 系统文件
.DS_Store
```

## 如果将来需要子包特殊配置

可以在子包添加.gitignore，但**仅用于该包特有的文件**：

```gitignore
# packages/some-package/.gitignore
# 仅该包特有的忽略规则
special-tool-cache/
package-specific-secret.key
```

**注意**：子包的.gitignore不会覆盖根目录的规则，而是叠加。
