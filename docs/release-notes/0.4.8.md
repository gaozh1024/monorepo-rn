# @gaozh1024/rn-kit 0.4.8 Release Notes

`0.4.8` 是一次 API 稳定性与可点击容器兼容性修正发布，主要补齐 `createAPI` 的异常兜底，以及 `AppPressable` 对原生 `Pressable` 状态样式的兼容。

## 本次更新

### 1. `createAPI` 路径参数缺失时改为提前失败

- 现在会继续检查 `{id}` / `:id` 这类路径占位符是否仍未被替换
- 如果缺少路径参数，会在真正发请求前直接抛出 `VALIDATION` 错误
- 避免把 `/users/{id}` 这类字面量 URL 误发到后端

### 2. 请求构建阶段错误统一纳入错误链路

现在以下阶段如果抛错，也会统一走：

- `onError`
- `AppError` 标准化
- `observability.error`

覆盖场景包括：

- `getHeaders()` 异步取 token 失败
- endpoint 级 `getHeaders()` 抛错
- 请求体序列化失败

### 3. `AppPressable` 保留原生 `style(state)` 语义

- 继续支持快捷参数：布局 / 间距 / 尺寸 / `bg` / `surface` / `rounded`
- 同时保留原生 `Pressable` 的 `style={({ pressed, hovered, focused }) => ...}` 用法
- Web / TV 等场景下的 `hovered`、`focused` 不再被静态丢失

### 4. 使用文档同步完善

- 框架 README 补充 `AppPressable` 状态样式示例
- API 使用说明补充路径参数缺失与请求构建错误的默认行为
- API 错误处理指南、开发态可观测性说明同步更新
